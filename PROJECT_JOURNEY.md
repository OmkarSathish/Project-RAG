# Project RAG - Development Journey

A comprehensive documentation of the project's evolution from initialization to production-ready distributed RAG system.

---

| # | Work Summary | Learning/Outcomes |
|---|--------------|-------------------|

| **2** | **Environment Configuration & API Key Management** - Created .env file structure for secure API key storage, integrated python-dotenv for environment variable loading, implemented configuration validation on startup | Understood security best practices for credential management, learned environment-based configuration patterns, discovered importance of validation before runtime |

| **5** | **OpenAI Embeddings Integration** - Integrated text-embedding-ada-002 model for document vectorization, implemented batch embedding to handle rate limits, added retry logic for API failures | Learned OpenAI embeddings API and rate limiting strategies, understood embedding dimensions (1536D for ada-002), gained experience with exponential backoff retry patterns |
| **6** | **Document Chunking with LangChain** - Implemented RecursiveCharacterTextSplitter with 1000 token chunks and 200 overlap, optimized chunk size for semantic coherence, preserved document metadata across chunks | Understood chunking tradeoffs (size vs context preservation), learned recursive splitting strategies (paragraphs → sentences → characters), discovered overlap importance for context continuity |

| **8** | **LangChain RAG Chain Construction** - Built basic RAG chain using RetrievalQA, integrated OpenAI GPT-3.5-turbo for answer generation, implemented prompt template for context injection | Understood RAG architecture (retrieval + generation), learned LangChain chain composition patterns, discovered prompt engineering for context-aware responses |
| **9** | **Basic Query Endpoint Development** - Created POST /query endpoint with JSON request/response, implemented error handling for invalid queries, added request logging for debugging | Learned RESTful API design principles, understood HTTP status code usage (200, 400, 500), gained experience with FastAPI request validation and Pydantic models |

| **11** | **BM25 Keyword Search Integration** - Added rank-bm25 library for sparse retrieval, implemented BM25Okapi indexing on document corpus, created keyword-based retrieval as complement to semantic search | Understood sparse vs dense retrieval methods, learned BM25 algorithm (TF-IDF extension with saturation), discovered when keyword search outperforms semantic search (exact terms, names) |
| **12** | **Hybrid Retrieval Implementation (Semantic + BM25)** - Implemented hybrid retriever combining vector similarity search with BM25 keyword matching, integrated Reciprocal Rank Fusion (RRF) algorithm with configurable weights (0.7 semantic, 0.3 keyword) | Understood the complementary nature of dense vs sparse retrieval methods, learned RRF fusion algorithm (rank-based scoring with k=60 constant), discovered hybrid retrieval improves recall by 15-30% over single method |
| **13** | **Cross-Encoder Reranker Development** - Integrated sentence-transformers cross-encoder (ms-marco-MiniLM-L-6-v2), implemented two-stage retrieval (retrieve 20 → rerank to 4), added reranking to hybrid retrieval pipeline | Learned bi-encoder vs cross-encoder tradeoffs (speed vs accuracy), understood two-stage retrieval benefits (fast initial retrieval + accurate final ranking), discovered cross-encoder computational costs |
| **14** | **Query Enhancement Pipeline** - Created query_enhancer.py with query expansion using LLM, implemented query reformulation for ambiguous inputs, added keyword extraction for better BM25 matching | Understood query understanding challenges (ambiguity, underspecification), learned LLM-based query expansion techniques, discovered importance of query preprocessing in retrieval quality |
| **15** | **Redis Caching Setup** - Added Redis container to docker-compose.yml, installed redis-py library with async support, configured connection pooling for efficient resource usage | Learned Redis fundamentals (key-value store, in-memory performance), understood connection pooling benefits (reduced overhead), gained experience with async Redis operations |
| **16** | **Query Result Caching Implementation** - Implemented cache key generation using query hashing (SHA-256), added cache lookup before RAG pipeline execution, set 5-minute TTL for cached results | Understood caching strategies for expensive operations, learned cache key design principles (collision avoidance, uniqueness), discovered TTL tradeoffs (freshness vs performance gains) |
| **17** | **Cache Hit/Miss Metrics** - Added Prometheus metrics for cache operations (hits, misses, hit rate), integrated prometheus-client library, created /metrics endpoint for scraping | Learned Prometheus metrics types (Counter, Gauge, Histogram), understood observability importance for caching effectiveness, discovered metric labeling strategies for dimensional analysis |
| **18** | **MongoDB Integration for Checkpointing** - Added MongoDB service to Docker Compose, integrated motor async MongoDB driver, implemented LangGraph checkpoint storage using langgraph-checkpoint-mongodb | Learned document database concepts (collections, documents, indexes), understood checkpoint persistence for conversation state, gained experience with async MongoDB operations (motor library) |
